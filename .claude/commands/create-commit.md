# commitメッセージを作成

## 実行手順

1. `git status`でファイルを確認
2. ステージング対象を適切な粒度に取り纏める
3. セキュリティ上問題のある情報が含まれてないか確認する
4. サイトに関係する実装内容且つ、リリース扱いで良さそうな内容であれば、CHANGELOG.mdに非エンジニア向けでもわかりやすい形にて変更点を日本語で記載する（インクリメントは基本、0.0.01ずつで、仕様書変更は含めない）
5. [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/)に従い、内容部分は日本語にてコミットを実行

## 形式の指定

- type(scope): subject の形式に従う
- タイトルは50文字以内、本文は72文字程度で改行
- 動詞は原形を使用（add, fix, updateなど）
- scope は原則記述するが、適切なものがない場合は省略可
- コミットメッセージは小文字で始める

## 実装とテストが含まれる場合の優先ルール

- 実装とテストコードが含まれている場合、typeはtestよりもfeat/fixを優先する
